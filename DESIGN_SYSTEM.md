# Design System Standard

## Principles

- Trust first: clear, stable, confidence-building UI
- Clarity over decoration
- Fast comprehension on mobile and desktop
- Accessibility as a default, not an afterthought

## Token Layers

- Foundation: color, type, spacing, radius, elevation
- Semantic: success/warning/error/info/neutral
- Component: variant-specific mappings

## Core Components

- Buttons (primary/secondary/tertiary/destructive)
- Inputs (text, textarea, select, phone, currency)
- Cards, tables, badges, alerts, modals, toasts
- Stepper/timeline for multi-step workflows

## Motion

- Use purposeful motion only (state transition, hierarchy, feedback)
- Duration range: 120ms to 280ms for most transitions
- Avoid ornamental animation on critical data-entry flows

## Accessibility

- WCAG AA contrast minimum
- Keyboard navigable interactive controls
- Visible focus states
- Correct semantic HTML + ARIA usage
- Error states must be announced and actionable

## Content and Tone

- Professional, direct, human
- No hype claims without proof
- Explicit uncertainty when confidence is low
- Compliance language on privacy/SMS pages must be precise
