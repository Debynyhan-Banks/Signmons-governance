# BE-001 - Keyword Opt-In/Out Persistence And Telemetry

## Ticket Summary

- Ticket ID: `BE-001`
- Repo: `signmons-calldesk-backend`
- Screen IDs: none (backend behavior ticket)
- Objective: enforce opt-in/opt-out keyword persistence and audit telemetry for messaging compliance.

## Scope

- Persist keyword opt state transitions (`START`, `STOP`, `UNSTOP`, `HELP`) per tenant + contact.
- Emit auditable telemetry on each state transition.
- Enforce fail-closed behavior when state cannot be resolved.

## Out of Scope

- UI screen implementation.
- Pricing/content changes.

## Acceptance Criteria

- Opt state is persisted and queryable by tenant/contact.
- Inbound keyword handling emits structured telemetry.
- Message-send paths respect opt state.
- Backend gates pass: build, tests, architecture checks.

## Evidence

- Automated test output for keyword flows.
- Sample telemetry/event payloads.
- Gate logs for required commands.
