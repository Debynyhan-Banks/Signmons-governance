# APP-011 - Customer Booking Status and Confirmation Flow

## Ticket Summary

- Ticket ID: `APP-011`
- Repo: `signmons-calldesk-backend` (+ frontend app surface)
- Screen IDs: `SCR-CUST-001`, `SCR-CUST-003`
- Objective: provide customer-facing booking status transparency and confirmation/reschedule workflow.

## In Scope

- Customer status view via secure link:
  - request received
  - pending/confirmed state
  - technician assignment status
  - payment state summary
- Customer actions:
  - confirm appointment window
  - request reschedule
- Dispatcher visibility of customer updates.

## Out of Scope

- Full customer account portal.
- Open self-service scheduling marketplace.
- Long-term customer profile management.

## Acceptance Criteria (must all pass)

- [ ] Customer can view booking status through secure link.
- [ ] Customer can confirm or request reschedule.
- [ ] Dispatcher sees customer response updates.
- [ ] Customer-facing actions are audit logged.
- [ ] Tenant boundaries are enforced.

## Test and Evidence Plan

- Backend gates: `npm run -s build`, `npm test -- --runInBand`, `npm run -s arch:check`
- Frontend gates (if touched): `npm run -s build`, `npm run -s lint`, `npm test -- --runInBand`
- Evidence:
  - customer status screenshots
  - confirm/reschedule flow proof
  - event/audit proof

## Risks

- Customer data exposure in link payloads.
- Reschedule conflicts with existing assignment flow.
- Notification timing confusion.

## Commit Convention

- Commit message: `feat(app): implement APP-011 customer booking flow`
- One focused commit only.
