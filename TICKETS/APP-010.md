# APP-010 - Routing Rules, Service Areas, and Availability

## Ticket Summary

- Ticket ID: `APP-010`
- Repo: `signmons-calldesk-backend` (+ frontend app surface)
- Screen IDs: `SCR-APP-015`, `SCR-APP-016`, `SCR-APP-024`, `SCR-TECH-004`
- Objective: implement tenant-configurable routing with service area, technician availability, and escalation policy controls.

## In Scope

- Technician schedule/availability and on-call controls.
- Routing rules for:
  - service category
  - urgency
  - service area
  - business hours / after-hours
  - escalation policy
- Service area and trade coverage management.
- Recommendation output with assignment reason trace.

## Out of Scope

- Free-form autonomous AI assignment without rule constraints.
- Multi-objective route optimization.
- Cross-tenant shared rule coupling.

## Acceptance Criteria (must all pass)

- [ ] Tenant can create/update routing and escalation rules.
- [ ] Service area and availability constraints are enforced.
- [ ] Emergency rules support owner + on-call escalation.
- [ ] Recommendation reason is visible in assignment workflow.
- [ ] Rule evaluations are audit logged.

## Test and Evidence Plan

- Backend gates: `npm run -s build`, `npm test -- --runInBand`, `npm run -s arch:check`
- Frontend gates (if touched): `npm run -s build`, `npm run -s lint`, `npm test -- --runInBand`
- Evidence:
  - routing rule and service-area screenshots
  - recommendation reasoning proof
  - escalation routing proof

## Risks

- Rule precedence conflicts.
- Unsafe defaults causing misrouting.
- Incomplete availability normalization.

## Commit Convention

- Commit message: `feat(app): implement APP-010 routing and availability`
- One focused commit only.
