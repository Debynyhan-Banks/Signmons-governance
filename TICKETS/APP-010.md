# APP-010 - Revenue Recovery and Conversion Dashboards

## Ticket Summary

- Ticket ID: `APP-010`
- Repo: `signmons-calldesk-backend` (+ frontend app surface)
- Screen IDs: `SCR-APP-018`, `SCR-APP-019`, `SCR-APP-020`
- Objective: expose owner-visible revenue and quality metrics that justify high-ticket operational value.

## In Scope

- Revenue recovery dashboard:
  - calls captured
  - missed calls recovered
  - jobs booked
  - deposits collected
  - estimated revenue recovered
- Conversion funnel analytics:
  - inbound -> qualified
  - qualified -> payment sent
  - payment sent -> paid
  - paid -> booked
  - booked -> completed
- AI quality dashboard:
  - repeat-question rate
  - first-response latency
  - escalation rate
  - intake failure rate
  - handoff rate
- Tenant/date-range filtering baseline.

## Out of Scope

- Predictive forecasting models.
- Multi-tenant benchmark rollups in tenant-facing dashboards.
- External BI warehouse integrations.

## Acceptance Criteria (must all pass)

- [ ] Revenue dashboard renders required recovery metrics.
- [ ] Funnel dashboard renders stage conversion rates.
- [ ] Quality dashboard renders latency/repeat/escalation metrics.
- [ ] Metrics are tenant-scoped and date-range filterable.
- [ ] Metric event lineage is traceable to auditable source events.

## Test and Evidence Plan

- Backend gates: `npm run -s build`, `npm test -- --runInBand`, `npm run -s arch:check`
- Frontend gates (if touched): `npm run -s build`, `npm run -s lint`, `npm test -- --runInBand`
- Evidence:
  - dashboard screenshots
  - metric calculation test output
  - tenant scoping proof

## Risks

- Metric drift between funnel stages and operational statuses.
- Missing event instrumentation for denominator values.
- Misleading estimates without transparent calculation notes.

## Commit Convention

- Commit message: `feat(app): implement APP-010 revenue recovery dashboards`
- One focused commit only.
