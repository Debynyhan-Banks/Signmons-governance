# APP-006 - Intake Review and Booking Readiness

## Ticket Summary

- Ticket ID: `APP-006`
- Repo: `signmons-calldesk-backend` (+ frontend app surface)
- Screen IDs: `SCR-APP-012`
- Objective: give owner/dispatcher a deterministic intake-readiness screen that confirms each job is dispatch-ready before assignment.

## In Scope

- Intake review screen with required booking fields:
  - customer name
  - phone
  - service address
  - service category
  - issue summary
  - urgency
  - preferred window
  - photos (if provided)
  - payment/deposit status
- Booking-readiness status:
  - ready to assign
  - missing info (with explicit missing-field list)
- Linked transcript/conversation trace from intake source.

## Out of Scope

- Technician assignment actions.
- Customer-facing update screens.
- Routing policy engine.

## Acceptance Criteria (must all pass)

- [ ] Intake review shows all required fields or explicit missing-field flags.
- [ ] Dispatcher can clearly determine booking-readiness state.
- [ ] Emergency/high-priority calls are visibly flagged.
- [ ] Tenant and role boundaries are enforced server-side.
- [ ] Readiness decisions are auditable.

## Test and Evidence Plan

- Backend gates: `npm run -s build`, `npm test -- --runInBand`, `npm run -s arch:check`
- Frontend gates (if touched): `npm run -s build`, `npm run -s lint`, `npm test -- --runInBand`
- Evidence:
  - intake review screenshots
  - missing-field scenario proof
  - API/read model proof for readiness state

## Risks

- Missing-field logic drift between channels.
- False positives for readiness when canonical fields are incomplete.
- Urgency context hidden from dispatcher.

## Commit Convention

- Commit message: `feat(app): implement APP-006 intake review readiness`
- One focused commit only.
