# APP-006 - Dispatch Board and Technician Assignment

## Ticket Summary

- Ticket ID: `APP-006`
- Repo: `signmons-calldesk-backend` (+ frontend app surface)
- Screen IDs: `SCR-APP-014`, `SCR-APP-017`
- Objective: provide dispatcher command-center workflows for assignment decisions with explicit recommendation rationale and auditability.

## In Scope

- Dispatch board with queues for new, ready, assigned, and escalated jobs.
- Job assignment detail with:
  - recommended assignee
  - assignment reason
  - manual override
  - assignment history
- Dispatcher actions:
  - assign
  - reassign
  - escalate
  - cancel
- Assignment event audit log integration.

## Out of Scope

- Full auto-dispatch mode.
- Route optimization.
- Technician mobile UX beyond assignment notification handoff.

## Acceptance Criteria (must all pass)

- [ ] Dispatcher can assign a ready job from the board.
- [ ] Assignment detail shows recommendation + reason.
- [ ] Dispatcher can override recommendation.
- [ ] Assignment and reassignment actions are audit logged.
- [ ] Tenant/RBAC checks enforced for all assignment endpoints.

## Test and Evidence Plan

- Backend gates: `npm run -s build`, `npm test -- --runInBand`, `npm run -s arch:check`
- Frontend gates (if touched): `npm run -s build`, `npm run -s lint`, `npm test -- --runInBand`
- Evidence:
  - dispatch board screenshot
  - assignment detail screenshot
  - API proof for assignment/reassignment
  - audit event proof

## Risks

- Dispatcher race conditions on simultaneous assignments.
- Ambiguous recommendation reasoning reducing trust.
- Missing escalation guardrails for emergency jobs.

## Commit Convention

- Commit message: `feat(app): implement APP-006 dispatch board assignment`
- One focused commit only.
