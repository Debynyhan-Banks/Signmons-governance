# APP-013 - Notification Center and Messaging Templates

## Ticket Summary

- Ticket ID: `APP-013`
- Repo: `signmons-calldesk-backend` (+ frontend app surface)
- Screen IDs: `SCR-APP-021`, `SCR-APP-022`, `SCR-TECH-005`
- Objective: centralize owner/dispatcher/technician/customer communication signals and template controls.

## In Scope

- Notification center for owner/dispatcher and technician events.
- Messaging template management for customer communications.
- Event categories:
  - intake created
  - urgency escalation
  - assignment update
  - payment status update
  - schedule confirmation/update
- Delivery status visibility (sent/failed/retry).

## Out of Scope

- External marketing campaign automation suite.
- Multi-channel social messaging orchestration.
- Advanced A/B experimentation framework.

## Acceptance Criteria (must all pass)

- [ ] Notification center displays role-appropriate operational events.
- [ ] Templates can be managed per tenant policy.
- [ ] Delivery status and failures are visible.
- [ ] Sensitive data is redacted per role boundaries.
- [ ] Notification events are auditable.

## Test and Evidence Plan

- Backend gates: `npm run -s build`, `npm test -- --runInBand`, `npm run -s arch:check`
- Frontend gates (if touched): `npm run -s build`, `npm run -s lint`, `npm test -- --runInBand`
- Evidence:
  - notification center screenshots
  - template configuration proof
  - failed-delivery handling proof

## Risks

- Template drift from policy/compliance constraints.
- Alert fatigue for dispatch roles.
- Overexposure of customer data in message previews.

## Commit Convention

- Commit message: `feat(app): implement APP-013 notification center templates`
- One focused commit only.
