# FE-001 - Marketing Home

## Ticket Summary

- Ticket ID: `FE-001`
- Repo: `Signmons-Marketing-Web-Feat-Marketing-Site`
- Screen IDs: `SCR-PUB-001`
- Objective: Ship a high-conversion homepage with clear value proposition, trust framing, and structured CTA flow.

## In Scope

- Home page route (`/`)
- Hero section with primary and secondary CTAs
- Value pillars section (3 cards)
- How-it-works section (5-step flow)
- Trust section (proof/safety claims grounded in real product behavior)
- CTA section (demo / early access)
- Responsive layout and accessibility baseline

## Out of Scope

- Full pricing page
- Full demo experience page
- Dashboard/app UI
- Auth or backend implementation changes

## Required Content Blocks

1. Hero:
- Headline focused on outcome and confidence
- Subheadline focused on bookings + clean confirmed data
- Primary CTA: `Book a Demo`
- Secondary CTA: `Request Early Access`

2. Value pillars:
- Answer Every Call
- Confirm Every Detail
- Payment Before Dispatch

3. How it works:
- Inbound call/text
- Intake and triage
- Confirmation via SMS
- Payment link
- Dispatch-ready job

4. Trust and safety:
- Fail-closed controls
- Human override
- Audit trail
- No false claims

5. CTA close:
- Repeat primary CTA and secondary CTA

## UX/Design Requirements

- Follow `DESIGN_SYSTEM.md`
- Mobile-first and desktop responsive
- Clear visual hierarchy and readable typography
- Accessible contrast and focus states
- No dark patterns, no fake counters/testimonials

## Technical Requirements

- Keep page implementation modular (section components)
- Avoid business logic in presentation-only components
- Keep static content data centralized for maintainability
- No coupling to backend auth or internal app routes

## Acceptance Criteria (must all pass)

- [ ] Home route renders all required content blocks
- [ ] Primary and secondary CTAs are visible in hero and close section
- [ ] Layout is responsive at common breakpoints (mobile/tablet/desktop)
- [ ] Lighthouse/accessibility spot check has no critical accessibility issue
- [ ] No references to unshipped features or unsupported guarantees
- [ ] Code follows repo lint/build standards
- [ ] Evidence captured (screenshots + command outputs)

## Evidence Required

- Screenshot set:
  - mobile full-page
  - desktop full-page
  - CTA section close-up
- Command output:
  - `npm run -s build`
  - `npm run -s lint`
  - test command for touched area (or explicit N/A note if no test harness)

## Risks

- Marketing copy over-promising capabilities not yet shipped
- Visual inconsistency with future product design tokens
- Accessibility regressions from animation-heavy sections

## Commit Convention

- Commit message: `feat(marketing): implement FE-001 home screen`
- One focused commit only for this ticket
