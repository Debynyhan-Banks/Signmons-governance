# APP-015 - AI Quality and Failure Review

## Ticket Summary

- Ticket ID: `APP-015`
- Repo: `signmons-calldesk-backend` (+ frontend app surface)
- Screen IDs: `SCR-APP-020`, `SCR-ADM-005`
- Objective: expose AI quality and failure diagnostics needed to operate a high-ticket, reliability-first dispatch platform.

## In Scope

- AI quality dashboard metrics:
  - repeat-question rate
  - first-response latency
  - escalation rate
  - intake failure rate
  - human handoff rate
- Admin failure review view for:
  - failed calls
  - failed intake transitions
  - failed webhook-linked outcomes
- Failure triage metadata and remediation status.

## Out of Scope

- Automated self-healing policy engine.
- Advanced model retraining pipeline.
- Full observability vendor replacement.

## Acceptance Criteria (must all pass)

- [ ] Quality dashboard displays required AI operational metrics.
- [ ] Failure review queue surfaces actionable incident details.
- [ ] Failures can be categorized and tracked to resolution.
- [ ] Tenant/admin access boundaries are enforced.
- [ ] Quality and failure events are auditable.

## Test and Evidence Plan

- Backend gates: `npm run -s build`, `npm test -- --runInBand`, `npm run -s arch:check`
- Frontend gates (if touched): `npm run -s build`, `npm run -s lint`, `npm test -- --runInBand`
- Evidence:
  - quality dashboard screenshots
  - failure queue screenshots
  - event lineage and audit proof

## Risks

- Metric interpretation ambiguity across teams.
- Incomplete failure context slows incident response.
- Alert noise obscuring severe failures.

## Commit Convention

- Commit message: `feat(app): implement APP-015 ai quality failure review`
- One focused commit only.
