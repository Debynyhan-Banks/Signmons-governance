# APP-014 - Revenue Recovery and Conversion Analytics

## Ticket Summary

- Ticket ID: `APP-014`
- Repo: `signmons-calldesk-backend` (+ frontend app surface)
- Screen IDs: `SCR-APP-018`, `SCR-APP-019`
- Objective: provide owner-facing metrics proving captured demand, conversion efficiency, and recovered revenue outcomes.

## In Scope

- Revenue recovery metrics:
  - calls captured
  - missed calls recovered
  - jobs booked
  - deposits collected
  - estimated revenue recovered
- Conversion funnel stages:
  - inbound -> qualified
  - qualified -> payment sent
  - payment sent -> paid
  - paid -> booked
  - booked -> completed
- Tenant and date-range filtering.

## Out of Scope

- Forecasting models.
- Cross-tenant benchmarking in tenant view.
- External BI export suites.

## Acceptance Criteria (must all pass)

- [ ] Revenue recovery dashboard displays required metrics.
- [ ] Conversion funnel displays stage conversion rates.
- [ ] Metrics are tenant-scoped and date-range filterable.
- [ ] Metric lineage is traceable to source events.
- [ ] Dashboard calculations are test-covered.

## Test and Evidence Plan

- Backend gates: `npm run -s build`, `npm test -- --runInBand`, `npm run -s arch:check`
- Frontend gates (if touched): `npm run -s build`, `npm run -s lint`, `npm test -- --runInBand`
- Evidence:
  - revenue/funnel dashboard screenshots
  - metric calculation tests
  - tenant-scoping proof

## Risks

- Stage-definition drift from operational statuses.
- Missing instrumentation for denominators.
- Misleading estimates without clear calculation notes.

## Commit Convention

- Commit message: `feat(app): implement APP-014 revenue conversion analytics`
- One focused commit only.
