# APP-008 - Dispatch Board and Technician Assignment

## Ticket Summary

- Ticket ID: `APP-008`
- Repo: `signmons-calldesk-backend` (+ frontend app surface)
- Screen IDs: `SCR-APP-014`, `SCR-APP-017`
- Objective: provide dispatcher command-center assignment workflows with recommendation rationale and safe manual override.

## In Scope

- Dispatch board queues:
  - new request
  - ready to assign
  - assigned
  - escalated
- Job assignment detail:
  - recommended tech
  - recommendation reason
  - assignment history
  - manual override controls
- Assignment actions:
  - assign
  - reassign
  - escalate
  - cancel

## Out of Scope

- Full autonomous auto-dispatch mode.
- Route optimization engine.
- Technician mobile UI beyond handoff signal.

## Acceptance Criteria (must all pass)

- [ ] Dispatcher can assign and reassign jobs from board/detail.
- [ ] Recommendation reason is visible before assignment.
- [ ] Manual override is supported for authorized users.
- [ ] Assignment actions are audit logged.
- [ ] Tenant and RBAC boundaries are enforced.

## Test and Evidence Plan

- Backend gates: `npm run -s build`, `npm test -- --runInBand`, `npm run -s arch:check`
- Frontend gates (if touched): `npm run -s build`, `npm run -s lint`, `npm test -- --runInBand`
- Evidence:
  - dispatch board and assignment detail screenshots
  - assignment/reassignment API proof
  - audit event proof

## Risks

- Race conditions on concurrent assignment updates.
- Poor recommendation explainability reduces trust.
- Escalation logic inconsistency by urgency level.

## Commit Convention

- Commit message: `feat(app): implement APP-008 dispatch assignment board`
- One focused commit only.
