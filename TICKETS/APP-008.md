# APP-008 - Customer Booking Status and Updates

## Ticket Summary

- Ticket ID: `APP-008`
- Repo: `signmons-calldesk-backend` (+ frontend app surface)
- Screen IDs: `SCR-CUST-001`, `SCR-CUST-003`, `SCR-CUST-004`
- Objective: establish customer-facing status transparency and update workflows so Signmons reliably closes the loop after intake.

## In Scope

- Customer booking status page via secure link:
  - request received
  - pending/confirmed state
  - technician assignment state
  - payment state
- Customer confirmation/reschedule action flow.
- Customer photo/notes upload support for job enrichment.
- Event logging for all customer state changes.

## Out of Scope

- Public account registration or customer portal login system.
- Full self-service scheduling engine.
- Review solicitation automation beyond status lifecycle handoff.

## Acceptance Criteria (must all pass)

- [ ] Customer can view current booking status through secure link.
- [ ] Customer can confirm or request reschedule.
- [ ] Customer can upload photos/notes when enabled by tenant policy.
- [ ] Dispatcher sees customer updates reflected in dispatch workflow.
- [ ] Customer update events are auditable.

## Test and Evidence Plan

- Backend gates: `npm run -s build`, `npm test -- --runInBand`, `npm run -s arch:check`
- Frontend gates (if touched): `npm run -s build`, `npm run -s lint`, `npm test -- --runInBand`
- Evidence:
  - customer status screen screenshots
  - confirm/reschedule flow proof
  - photo upload policy enforcement proof

## Risks

- Exposing too much customer/job data in unauthenticated links.
- Reschedule edge cases conflicting with dispatcher assignments.
- Upload controls missing limits/validation.

## Commit Convention

- Commit message: `feat(app): implement APP-008 customer booking updates`
- One focused commit only.
