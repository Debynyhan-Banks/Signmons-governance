# APP-009 - Technician Mobile Job Workflow (PWA)

## Ticket Summary

- Ticket ID: `APP-009`
- Repo: `signmons-calldesk-backend` (+ frontend app surface)
- Screen IDs: `SCR-TECH-001`, `SCR-TECH-002`, `SCR-TECH-003`
- Objective: provide technician-facing, mobile-first secure-link workflows to accept and progress assigned jobs.

## In Scope

- Technician job list (today, upcoming, completed).
- Technician job detail with key job/customer fields.
- Technician status actions:
  - accept
  - decline
  - on my way
  - in progress
  - complete
  - cannot take job
- Secure signed-link access model with expiration.

## Out of Scope

- Native app-store release.
- Payroll or HR features.
- Route optimization/navigation integration.

## Acceptance Criteria (must all pass)

- [ ] Technician can open secure link and view assigned jobs only.
- [ ] Technician can update job state from mobile view.
- [ ] Status updates reflect in dispatcher workflows.
- [ ] Expired/invalid links fail closed.
- [ ] Tenant and RBAC protections are enforced server-side.

## Test and Evidence Plan

- Backend gates: `npm run -s build`, `npm test -- --runInBand`, `npm run -s arch:check`
- Frontend gates (if touched): `npm run -s build`, `npm run -s lint`, `npm test -- --runInBand`
- Evidence:
  - technician list/detail screenshots
  - secure-link validation proof
  - status propagation proof

## Risks

- Token leakage if signed-link scope is too broad.
- Status lifecycle mismatch between tech and dispatcher views.
- Notification timing inconsistencies.

## Commit Convention

- Commit message: `feat(app): implement APP-009 technician mobile workflow`
- One focused commit only.
