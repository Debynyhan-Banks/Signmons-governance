# APP-009 - Routing Rules and Service-Area Coverage

## Ticket Summary

- Ticket ID: `APP-009`
- Repo: `signmons-calldesk-backend` (+ frontend app surface)
- Screen IDs: `SCR-APP-016`, `SCR-APP-024`
- Objective: provide tenant-configurable dispatch rules for category, urgency, service area, schedule, and escalation with deterministic assignment recommendations.

## In Scope

- Routing rule configuration for:
  - category/trade match
  - urgency level
  - service area
  - business hours / after-hours
  - on-call escalation
- Service-area and trade coverage management.
- Assignment recommendation output with reason trace.
- Tenant-isolated policy storage and evaluation.

## Out of Scope

- AI free-form autonomous assignment without rules.
- Multi-objective route optimization engine.
- Cross-tenant shared rule templates with live coupling.

## Acceptance Criteria (must all pass)

- [ ] Tenant can create/update routing rules by category/urgency/area.
- [ ] Rules produce deterministic recommendation + reason.
- [ ] Emergency rules support owner + on-call escalation.
- [ ] Service-area coverage is enforced during recommendation.
- [ ] Rule evaluation and assignment recommendations are audit logged.

## Test and Evidence Plan

- Backend gates: `npm run -s build`, `npm test -- --runInBand`, `npm run -s arch:check`
- Frontend gates (if touched): `npm run -s build`, `npm run -s lint`, `npm test -- --runInBand`
- Evidence:
  - rule configuration screenshots
  - recommendation reason output proof
  - emergency escalation rule proof

## Risks

- Conflicting rules with unclear precedence.
- Overly permissive defaults causing unsafe assignment.
- Incomplete service-area normalization.

## Commit Convention

- Commit message: `feat(app): implement APP-009 routing rules and service areas`
- One focused commit only.
