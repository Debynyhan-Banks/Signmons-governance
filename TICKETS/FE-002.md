# FE-002 - Public Trust/Compliance Screens

## Ticket Summary

- Ticket ID: `FE-002`
- Repo: `Signmons-Marketing-Web-Feat-Marketing-Site`
- Screen IDs: `SCR-PUB-005`, `SCR-PUB-010`
- Objective: ship public-facing trust/compliance pages that reduce buyer risk and satisfy messaging/legal disclosure expectations.

## In Scope

- `SCR-PUB-005`: `/trust` page
  - Trust/safety/compliance hero
  - Core controls section (explicit confirmations, fail-closed, auditability, consent, human override)
  - Operational compliance notes
  - FAQ section for skeptical buyers
- `SCR-PUB-010`: `/legal` page
  - Privacy policy summary
  - Terms and conditions summary
  - SMS program terms including HELP/STOP behavior
- Responsive layout and accessibility baseline
- CTA continuity (`Try Demo`, `Join Early Access`)

## Out of Scope

- SaaS authenticated dashboard pages
- Backend legal policy engines
- External legal CMS integration
- Jurisdiction-specific legal advice customization

## Technical Requirements

- Centralize trust/legal copy in typed data contracts
- Keep route selection in routing layer only
- Keep page components presentation-focused (no backend coupling)
- Reuse existing marketing layout/system components

## Acceptance Criteria (must all pass)

- [ ] `/trust` renders the required trust/safety/compliance sections

- [ ] CTAs are visible and functional on both pages
- [ ] Mobile and desktop layouts are readable and responsive
- [ ] No unsupported guarantees or fabricated security claims
- [ ] Frontend gates executed with evidence:
  - `npm run -s build`

## Evidence Required

- Build/lint/test command output logs
