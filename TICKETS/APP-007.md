# APP-007 - Technician Mobile Job View (PWA)

## Ticket Summary

- Ticket ID: `APP-007`
- Repo: `signmons-calldesk-backend` (+ frontend app surface)
- Screen IDs: `SCR-TECH-001`, `SCR-TECH-002`, `SCR-TECH-003`
- Objective: deliver secure technician job workflows via mobile-first PWA links instead of native app-store clients.

## In Scope

- Technician job list view (today/upcoming/completed).
- Technician job detail with core fields:
  - customer
  - address
  - issue summary
  - urgency
  - preferred window
  - payment/deposit status
- Status updates:
  - accept
  - decline
  - on my way
  - in progress
  - complete
  - cannot take job
- Secure signed link access model with expiration.

## Out of Scope

- Native iOS/Android app packaging.
- Technician payroll/timekeeping.
- Route navigation optimization.

## Acceptance Criteria (must all pass)

- [ ] Tech can open secure link and view only assigned jobs.
- [ ] Tech can update job status from detail screen.
- [ ] Status updates propagate to dispatcher view.
- [ ] Expired/invalid links fail closed.
- [ ] Tenant isolation and role boundaries are enforced server-side.

## Test and Evidence Plan

- Backend gates: `npm run -s build`, `npm test -- --runInBand`, `npm run -s arch:check`
- Frontend gates (if touched): `npm run -s build`, `npm run -s lint`, `npm test -- --runInBand`
- Evidence:
  - mobile screenshots for tech list + detail
  - signed-link validation proof
  - status change propagation proof

## Risks

- Link sharing risk if token scope/expiry is too broad.
- Incomplete status lifecycle mapping.
- Notification lag causing technician confusion.

## Commit Convention

- Commit message: `feat(app): implement APP-007 technician mobile pwa view`
- One focused commit only.
