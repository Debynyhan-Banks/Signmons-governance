# APP-007 - Urgency Classification and Escalation Review

## Ticket Summary

- Ticket ID: `APP-007`
- Repo: `signmons-calldesk-backend` (+ frontend app surface)
- Screen IDs: `SCR-APP-013`
- Objective: make urgency decisions explainable, reviewable, and safely overridable for owner/dispatcher workflows.

## In Scope

- Urgency review view for:
  - emergency
  - high priority
  - standard
- Classification rationale:
  - trigger details
  - model/rule confidence notes
  - escalation path preview
- Authorized override workflow with reason capture.
- Escalation event visibility (who was notified, when, and outcome).

## Out of Scope

- Full dispatch assignment controls.
- Technician status updates.
- Customer self-service scheduling.

## Acceptance Criteria (must all pass)

- [ ] Urgency level is visible on each reviewed job.
- [ ] Rationale for urgency classification is visible to authorized roles.
- [ ] Authorized users can override urgency with mandatory reason.
- [ ] Escalation notifications and overrides are audit logged.
- [ ] Tenant and RBAC controls are enforced.

## Test and Evidence Plan

- Backend gates: `npm run -s build`, `npm test -- --runInBand`, `npm run -s arch:check`
- Frontend gates (if touched): `npm run -s build`, `npm run -s lint`, `npm test -- --runInBand`
- Evidence:
  - urgency review screenshots
  - override path proof
  - escalation event audit proof

## Risks

- Overriding urgency without clear authorization boundaries.
- Escalation fatigue from noisy classification.
- Missing rationale trace reduces operator trust.

## Commit Convention

- Commit message: `feat(app): implement APP-007 urgency escalation review`
- One focused commit only.
