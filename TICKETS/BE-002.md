# BE-002 - Marketing Lead-Capture API Persistence Contract

## Ticket Summary

- Ticket ID: `BE-002`
- Repo: `signmons-calldesk-backend`
- Screen IDs: `SCR-PUB-009`
- Objective: implement persisted contact lead capture for FE-007 with contract-tested API behavior.

## In Scope

- Implement `POST /api/marketing/lead-capture`.
- Validate email-minimum and consent fields.
- Persist lead in `marketingLead` model.
- Return deterministic success/error payloads.
- Emit audit/log events for accepted and rejected submissions.

## Out of Scope

- CRM outbound sync.
- Multi-step nurture campaign logic.
- UI changes in marketing repo.

## Acceptance Criteria

- [ ] Endpoint exists and is protected by input validation.
- [ ] Missing/invalid email returns `400` with structured error response.
- [ ] Valid requests persist lead and return `leadId` + `status` + `createdAt`.
- [ ] Contract shape matches `DATA_CONTRACTS.md` FE-007 section.
- [ ] Backend gates pass:
  - `npm run -s build`
  - `npm test -- --runInBand`
  - `npm run -s arch:check`

## Tests Required

- Controller test: accepts valid payload and returns success shape.
- Controller test: rejects missing/invalid email.
- Service test: persists required `marketingLead` fields.
- Contract test: response keys and data types are stable.

## Evidence Required

- Build/test/arch-check logs.
- Example request/response payloads.
- Persistence proof (test-level or integration log output).
